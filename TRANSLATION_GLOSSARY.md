# TON 白皮書台灣繁體中文翻譯術語表

本文件定義 TON 技術文件中所有關鍵術語的標準繁體中文翻譯。

譯者：Dr. Awesome Doge

## 翻譯原則

1. **技術準確性優先**：確保技術概念準確傳達
2. **保留專有名詞**：TON、TVM、Catchain 等保持原文
3. **符合台灣用語習慣**：使用台灣通用的技術術語
4. **一致性**：相同術語在所有文件中使用統一翻譯
5. **可讀性**：避免生硬直譯，保持文句流暢

## 特殊術語說明

### 1. Catchain
**翻譯**：Catchain（保留原文）

**說明**：這是 TON 共識協議的專有名稱，源於 "catch-chain"（捕捉鏈），指專門用來捕捉共識過程中所有重要事件的特殊區塊鏈。由於是專有名詞且廣為人知，建議保留原文。

### 2. Cell
**翻譯**：單元 / Cell

**說明**：TON 區塊鏈中的基本資料結構。第一次出現時可標註為「單元（Cell）」，後續可視情況使用中文或保留原文。

### 3. Gas
**翻譯**：Gas / 燃料

**說明**：在區塊鏈領域，Gas 已成為通用術語。建議保留原文，必要時可註明「（計算資源費用）」。

### 4. Smart Contract
**翻譯**：智慧合約

**說明**：台灣慣用「智慧」而非中國大陸的「智能」。

## 文體風格指引

### 1. 語氣
- 使用專業但易懂的語氣
- 避免過於口語化或生硬的直譯
- 保持客觀、中立的學術風格

### 2. 句式
- 優先使用主動語態
- 適度使用被動語態（when appropriate）
- 保持句子結構清晰

### 3. 標點符號
- 使用全形中文標點：，。；：「」『』（）
- 英文、數字前後加空格（除非緊鄰標點）
- 專有名詞前後適度加空格以提高可讀性

### 4. 數字與單位
- 阿拉伯數字：用於技術規格、版本號
- 中文數字：用於一般敘述（如「三個步驟」）
- 單位保持原文：KB, MB, GB, ms, etc.
